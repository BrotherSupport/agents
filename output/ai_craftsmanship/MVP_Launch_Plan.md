# AI 數位工匠 (AI Craftsmanship) - MVP 30 天啟動計畫

## 1. 計畫概述
本計畫旨在 30 天內完成「AI 數位工匠」品牌的市場切入，從建立數位實績（Showcase）到大規模市場掃蕩（Ground Sweep），最終實現標準化運營（Standardized Ops）。

*   **目標市場**：台灣中小企業 (SMB)，重點鎖定物流業、製造業。
*   **核心價值**：數位土木工程 (Digital Civil Engineering) —— 節省工時、填補技術斷層。
*   **品牌定位**：您的 24/7 不休息「數位員工」。

---

## 2. 階段性任務與時程

### 第一階段：品牌展示期 (Day 1-7: Showcase)
**目標**：建立品牌可信度，產出具備「即視感」的示範案例。
*   **關鍵里程碑**：完成 3 份產業 Demo 與銷售簡報。
*   **代理人任務分配**：
    *   **Content Generation Agent (內容生成)**：撰寫 3 篇針對「金屬加工良率」與「物流單據自動化」的成功案例研究（Case Studies）。
    *   **Marketing Campaign Agent (行銷活動)**：完成品牌 Landing Page 內容設計，強調「數位工匠」的專業感。
    *   **Solution Design Agent (方案設計)**：構建兩套標準化 POC 原型（如：自動報價 AI、庫存預測 AI）供展示使用。
    *   **Lead Intelligence Agent (潛在客戶情報)**：初步篩選 50 家具備「AI 導入潛力」的台灣製造業與物流業名單。

### 第二階段：市場掃蕩期 (Day 8-21: Ground Sweep)
**目標**：大規模觸達潛在客戶，獲取「實地勘查 (Field Survey)」訂單。
*   **關鍵里程碑**：獲得至少 5 個實地勘查預約。
*   **代理人任務分配**：
    *   **Cold Email Automation Agent (開發信自動化)**：啟動針對 Top 50 名單的個人化開發信序列，採用「解決缺工」為切入點。
    *   **SEO & Visibility Agent (搜尋曝光)**：在 LinkedIn 與在地社團發布「數位轉型降本增效」系列短文，吸引有機流量。
    *   **Sales Conversion Agent (銷售轉化)**：處理郵件回覆，自動排定 15 分鐘線上初步諮詢，並轉化為 $18,000 TWD 的「實地勘查」合約。
    *   **Industry Specialization Agents (產業專家)**：在銷售通話中提供專業深度建議，確保客戶感受「懂行」。

### 第三階段：標竿運營期 (Day 22-30: Standardized Ops)
**目標**：標準化交付流程，將勘查轉化為 $58,000 TWD 的 POC 專案。
*   **關鍵里程碑**：簽署首 2 件 POC 合約，完成自動化交付 SOP。
*   **代理人任務分配**：
    *   **Client Onboarding Agent (客戶入職)**：啟動首批客戶的「數位脆弱點」診斷流程。
    *   **Strategy Assessment Agent (策略評估)**：產出具備 ROI 試算的「AI 導入建議書」。
    *   **POC Development Agent (原型開發)**：標準化「14 天快速原型」開發路徑，確保交付速度。
    *   **Financial Operations Agent (財務營運)**：建立自動化帳單發送與在地支付（金流）對帳系統。
    *   **Project Management Agent (專案管理 - 本人)**：優化跨代理人協作邏輯，彙整首月數據進行下月調整。

## 3. 成功指標 (Success Metrics)
| 指標項目 | 目標值 (Target) | 說明 |
| :--- | :--- | :--- |
| **潛在客戶獲取 (Leads)** | 50+ 筆 | 精準篩選的台灣 SMB 名單。 |
| **轉換率 (Conversion)** | > 10% | 從開發信到諮詢通話的轉化。 |
| **勘查訂單 (Field Surveys)** | 5+ 件 | 每件價值 $18,000 TWD。 |
| **POC 轉化 (POC Wins)** | 2+ 件 | 從勘查轉化為 $58,000 TWD 專案。 |
| **內容產出 (Content)** | 8+ 篇 | 包含案例、文章與技術簡報。 |

---

## 4. 風險管理
*   **技術風險**：POC 效果不如預期。
    *   *對策*：由 **POC Development Agent** 預先建立模組化組件，縮短除錯時間。
*   **信任風險**：傳統企業對 AI 安全性的疑慮。
    *   *對策*：**Strategy Assessment Agent** 在建議書中加入「資料地端處理」與「隱私保護」專章。
